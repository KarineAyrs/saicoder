## Tool `check` output format

After writing source files you should call tool `check` to check if your code is valid.
This tool returns JSON with potential mistakes written in it.

JSON response has the following schema:
- overall_evaluation: evaluation of the compliance of the code with the user query
- mistakes: list of mistakes (if any)
    - file_path: path to a file where error occurred
    - line: line number where error occurred
    - quote: exact quote of erroneous line
    - reason: explanation why this line is considered erroneous

You should analyze mistakes and rewrite code correctly.
If the `check` tool returned empty list, then you are doing great, and you should finish execution.
Otherwise, review the errors and fix the code, then call `check` tool for another review round.
